# QArcade License Audit

Status: preparatory engineering audit; not legal advice.  
Audit date: 2026-07-17.

## Executive summary

QArcade currently combines reusable framework code, educational examples,
commercially relevant games and content, hardware documentation, brand assets,
and third-party code. A repository-wide blanket license would obscure those
boundaries and could purport to license material QArcade does not exclusively
own.

The safe initial model is an explicit allowlist: only the reusable Qilin
framework and named development infrastructure enter the Apache-2.0 code
scope. Historical games, examples, content, hardware, and brand assets remain
outside that scope until ownership and provenance are reviewed.

## Framework boundary

The Qilin framework documentation identifies these as framework-owned:

- MicroQiskit simulation and quantum circuit compilation;
- Controller Grid and gate/input behavior;
- Key Map and Operation Feedback;
- layout parsing and preview tooling.

It identifies levels, Mission canvas content, scoring/progression, Response visualization,
game mechanics, and completion experiences as game-owned. Because the current
authoritative cartridge contains both categories, its Apache-2.0 designation
must be read together with `LICENSE_SCOPE.md`: third-party MicroQiskit stays
under upstream terms and game-owned creative content remains reserved until a
clean engine/content boundary is legally and technically confirmed. No source
refactor is part of this licensing-preparation change.

## Third-party inventory and obligations

### MicroQiskit / Qiskit

The upstream MicroQiskit repository identifies Apache-2.0 as its license.
Local copies carry IBM 2020/Qiskit notices and contain a small routine credited
as adapted from Stack Overflow. Redistribution should include Apache-2.0,
retain copyright/attribution notices, mark material changes, and verify the
specific Stack Overflow post's applicable license and attribution format.

### PICO-8

PICO-8 is a commercial platform dependency. QArcade may license contributor-
owned cartridge source, but cannot relicense the PICO-8 application, runtime,
font, trademarks, or platform assets. Exported HTML/JavaScript must be reviewed
under the export version's terms. The CC0 statement in the qShooter HTML shell
does not cover injected cartridge code or artwork.

### QPong and cited sources

`qpong.p8` expressly cites upstream QPong projects, IBM/Qiskit code, a Stack
Overflow answer, and a Lexaloffle BBS source. Do not redistribute it as a
QArcade-proprietary or Apache-2.0 work until source versions, licenses,
modifications, and author permissions have been recorded.

### Pillow and bundled libraries

Qilin preview tooling declares `Pillow>=10.0`. Releases that vendor Pillow or
a virtual environment must reproduce the notices for the installed Pillow
version and bundled libraries. Local `.venv` and cache directories are ignored
development artifacts and should not enter releases.

### Retro8 font source

The preview renderer fetches Retro8 `pico_font.h` at runtime. Current policy
correctly avoids committing or packaging the downloaded font. Review upstream
terms before any future vendoring.

### GitHub Actions

The Qilin workflow uses `actions/checkout@v4` and `actions/setup-python@v5`.
They are external build dependencies and are not relicensed by QArcade.

## Ownership questions still open

Before broadening the license scope, obtain a written record of:

1. the legal name and copyright years for QArcade and `wslu42/qilin`;
2. contributor consent for each named historical game directory;
3. whether student work was produced under university, event, employment, or
   sponsorship terms affecting ownership;
4. provenance of cartridge sprites, maps, sound effects, music, and text;
5. exact upstream revisions used for MicroQiskit, QPong, and exported PICO-8
   files; and
6. ownership of the QArcade logo and externally hosted README images.

## Classroom readiness

The repository is suitable for instructor-led viewing and demonstration after
the scope notices are present. For student modification or redistribution,
prefer an engine-only classroom package containing only verified Apache-2.0
paths, the applicable license, and NOTICE. The mixed historical repository
should not be described as wholly open source.
